# Contributing to HiShare

Thank you for your interest in contributing! HiShare is designed as an open-source health data platform, and contributions are welcome from designers, developers, writers, and accessibility advocates.

## How to contribute

1. Fork the repository.
2. Create a new branch for your feature or fix.
3. Make your changes.
4. Submit a pull request with a clear title and description.

## Code guidelines

- Use TypeScript for new React components.
- Follow the existing Tailwind CSS style patterns.
- Keep components reusable and accessible.
- Write small, focused commits.

## Issue workflow

- Choose an issue from [ISSUES.md](./ISSUES.md) or open a new one.
- Label your work clearly.
- If you begin an issue, leave a comment so maintainers know it is in progress.

## Styling and UI

- Use Tailwind classes for spacing, layout, and color.
- Keep the user interface clean, consistent, and patient-centered.
- Prefer semantic HTML and accessible form controls.

## Reporting bugs

- Reproduce the issue if possible.
- Include steps to reproduce, expected behavior, and actual behavior.
- Attach screenshots or a short screen recording if relevant.
